---

## 二、已妥善处理

| 检查项 | 涉及位置 | 处理方式 |
|--------|---------|---------|
| 密码加密存储 | SysUserServiceImpl.register() | BCrypt 加密，不存储明文。 |
| JWT签名密钥 | application.yml jwt.secret | 配置文件中独立管理，非硬编码在代码中。 |
| SQL注入防护 | MyBatis-Plus LambdaQueryWrapper | 使用 Lambda 表达式构建查询条件，非字符串拼接。 |
| 角色权限控制 | SecurityConfig.java | 3级角色（GUEST/FRONT_DESK/ADMIN）URL级别拦截。 |
| 登录防暴力破解 | LoginRateLimiter.java | IP+用户名维度5次失败锁定15分钟。 |
| 文件上传类型校验 | UploadController.java | Content-Type + 后缀名双重白名单校验。 |
| 文件上传大小限制 | application.yml + UploadController | 5MB限制，前后端双重校验。 |
| UUID防路径遍历 | UploadController.java | 上传文件重命名为随机UUID，防止覆盖和路径遍历。 |
| CORS配置 | CorsConfig.java | 跨域请求允许，但限制方法和暴露头。 |
| 逻辑删除 | MyBatis-Plus @TableLogic | 删除操作软删除，数据可恢复。 |

---

## 三、需关注

| 检查项 | 风险等级 | 说明 |
|--------|---------|------|
| JWT密钥强度 | 低 | 当前密钥为演示用固定字符串，生产环境应使用环境变量或密钥管理服务。 |
| 限流为内存方案 | 低 | LoginRateLimiter 基于 ConcurrentHashMap，应用重启后计数清零。对于单机部署可接受。 |
| 文件上传目录可预测 | 低 | /uploads/ 目录当应用部署在公网时需确保 Web 服务器正确配置了静态资源访问权限。 |
| 日志敏感信息 | 低 | 审计日志中的 requestParams 记录完整请求参数，需注意不要记录明文密码。当前注册接口密码字段已加密，影响可控。 |
| H2 Console | 低 | dev 环境下启用 H2 Console (/h2-console)，prod 环境下应确认关闭。当前 prod 配置中未开启 H2。 |

---

## 四、建议修复

无高优先级安全问题需要修复。当前安全措施已覆盖中小型酒店管理系统的常见安全需求。

## 五、总体评价

项目安全意识较好，核心安全路径（认证、鉴权、密码存储、SQL注入、文件上传、登录防护）均有对应的防护措施。综合风险等级：低。

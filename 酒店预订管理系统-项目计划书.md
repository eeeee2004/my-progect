# 项目计划书

## 酒店预订管理系统

---

## 一、总体计划

本项目拟采用分版本迭代开发策略，从基础预订功能起步，逐步增加分页、审计、安全、定价、可视化等高级功能，最终形成功能完整、体验优良的中小型酒店预订管理系统。开发方式为单人全栈迭代开发，每个版本聚焦一个核心功能领域。

---

## 二、项目范围说明书

### 2.1 范围基准

项目交付范围包括：

- **后端服务**：Spring Boot 应用，含 40+ REST API 接口，7 张数据库表。
- **前端应用**：Vue 3 单页应用，含 20+ 页面组件，3 个角色端。
- **数据库**：MySQL 模式定义（schema.sql）+ 种子数据（data.sql）。
- **项目文档**：14 份项目文档（建议书、可行性报告、需求说明书、计划书、Agent文档、开发日志、版本日志、质量计划书、质量检查报告、代码评审报告、安全审查报告、Readme、测试报告、验收报告）。

### 2.2 项目范围管理计划

新增需求需在每周迭代规划时评估。非紧急需求放入后续版本，紧急缺陷可插入当前版本。需求变更需更新对应的需求规格说明书。

## 六、效益分析

- **直接效益**：为中小酒店提供一个可自部署的免费管理系统，替代昂贵的商业PMS。
- **间接效益**：数据积累为智能定价、需求预测等高级功能提供基础。
- **学习价值**：作为全栈项目开发案例，涵盖前后端、数据库、安全等多个技术领域。

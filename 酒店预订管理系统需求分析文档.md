# 酒店预订管理系统需求分析说明书



**文档版本**：V1.1**适用课程**



**开发技术栈**：Vue3（前端）+ SpringBoot（后端）+ MySQL 8.0（数据库）**开发工具**



**架构模式**：前后端分离 B/S 架构



## 1 项目引言



### 1.1 编写目的



传统中小型酒店普遍采用纸质登记、Excel 台账、电话人工预订的管理模式，存在房态混乱、重复订房、账目统计繁琐、客户信息易丢失、人力成本高、服务效率低等痛点。



本需求分析说明书**基于 Vue + SpringBoot + MySQL 技术栈**，明确系统开发目标、用户角色、业务流程、功能需求、非功能需求、数据库设计及项目开发约束，为使用 Trae CN 进行前后端代码开发、系统设计、测试验收、课程答辩提供完整标准化依据，实现酒店客房、预订、入住退房、后台统计全流程信息化管理。



### 1.2 项目背景



当下中小型酒店缺乏轻量化数字化管理系统，人工管理模式弊端突出：房态靠人工记忆易出现重单漏单；营收、入住率依赖人工统计，误差大、效率低；无线上自主预订渠道，客源受限；员工无权限划分，数据安全性差。



本系统采用**前后端分离 B/S 架构**，前端使用 Vue3 开发页面交互，后端基于 SpringBoot 提供接口服务，MySQL 存储业务数据，无需安装客户端，浏览器即可访问，适配 Trae CN 一站式开发，为中小型酒店提供低成本、易开发、易部署的数字化解决方案。



### 1.3 文档适用范围



本文档覆盖系统可行性分析、用户角色、核心业务流程、前后端功能模块、非功能性需求、数据库实体设计、开发环境与项目约束，可直接作为课程项目报告、开发依据、答辩文档使用。

## 3 用户角色与核心业务流程



### 3.1 用户角色定义



系统采用**多角色权限隔离**设计，不同角色权限独立，后端 SpringBoot 做权限拦截，前端 Vue 控制页面路由访问：



1. **普通住客**：注册登录、浏览客房、在线预订、查看个人订单、维护个人资料；
2. **前台工作人员**：查看实时房态、办理入住退房、管理预订订单、登记散客信息、查询客户入住记录；
3. **系统管理员**：管理房型与房间信息、维护员工账号与权限、查看运营统计报表、系统基础数据维护。



### 3.2 核心业务流程



#### 3.2.1 用户在线预订流程



用户注册 / 登录系统 → 选择入住、离店日期 → 系统自动筛选空闲客房 → 浏览房型详情、价格及配套设施 → 填写入住人信息 → 提交预订订单 → 后端锁定房间防止重复预订 → 生成订单并展示订单状态。



#### 3.2.2 客人入住办理流程



客人到店出示预订订单 / 散客现场登记 → 前台核对身份信息 → 分配具体房间 → 录入押金信息 → 系统更新房态为**已入住** → 保存入住记录。



#### 3.2.3 客人退房结算流程



前台发起退房操作 → 后端自动计算住宿天数、房费及额外杂费 → 生成结算账单 → 客人确认缴费 → 系统完成退房结算 → 房态更新为**待清洁**，清洁后恢复为**空闲**。



#### 3.2.4 后台管理流程



管理员登录后台 → 维护房型、房间基础数据 → 新增 / 编辑前台员工账号并分配权限 → 查看每日 / 每月入住率、营业额报表 → 维护系统基础配置与异常数据处理。



## 4 功能需求分析



系统分为**住客客户端、前台管理端、系统管理员端**三大模块，采用前后端分离开发，Vue 负责页面展示与交互，SpringBoot 负责业务逻辑与数据处理。



### 4.1 住客客户端功能（Vue 前端实现）



1. **账号注册与登录**：支持用户名 / 手机号注册、密码登录，实现密码找回功能；
2. **客房查询与展示**：按入住离店日期筛选可订房间，展示房间图片、房型、价格、床位及配套设施；
3. **在线预订管理**：填写入住人姓名、手机号、证件信息提交预订；实时查看订单状态（待入住、已完成、已取消），支持未入住订单自主取消；
4. **个人中心**：修改个人基础资料、查询历史所有预订订单与入住记录。



### 4.2 前台管理端功能



1. **房态实时看板**：可视化色块展示所有房间状态，包含空闲、已预订、已入住、待清洁、维修中，支持一键刷新房态；
2. **预订订单管理**：查看当日预抵、预离客人列表；处理线上用户预订请求，支持手动添加线下散客预订；
3. **入住退房办理**：支持有订单一键入住、无订单散客快速登记；退房自动核算房费与总金额，生成结算账单；
4. **客户信息管理**：按姓名、手机号检索住客历史入住记录，建立简易客户档案。



### 4.3 系统管理员端功能



1. **客房信息管理**：新增、修改、删除房间信息，维护房间号、楼层、关联房型、房间状态；
2. **房型管理**：配置标准间、大床房、豪华套房等房型，设置房型简介、配套设施、基础价格；
3. **员工权限管理**：添加前台员工账号、重置密码、分配操作权限、禁用离职员工账号；
4. **数据统计报表**：按日、按月统计入住率、客房空置率、总营业额、订单数量，支持数据可视化展示；
5. **系统基础维护**：基础业务参数配置、异常订单处理、基础数据备份。



## 5 非功能性需求



### 5.1 性能需求



1. 系统页面响应时间不超过 2 秒，客房查询、订单加载、房态刷新无明显卡顿；
2. 支持至少 20 个并发用户同时在线操作，满足课程演示及小型酒店日常使用；
3. 前后端接口数据传输稳定，房态、订单数据实时同步无延迟。



### 5.2 安全性需求



1. 用户、员工密码采用 BCrypt 加密存储，禁止明文保存，保护敏感信息；
2. 基于角色实现权限控制，普通住客无法访问后台管理页面，前台无系统配置权限；
3. 并发预订时后端加锁控制，杜绝同一时间段同一房间重复预订；
4. 关键操作（取消订单、删除数据、权限修改）保留操作记录，可追溯。



### 5.3 可靠性需求



1. 保证数据库数据一致性，避免并发操作导致数据错乱、重单漏单；
2. 系统运行稳定，无频繁崩溃、页面报错、接口异常问题；
3. 核心业务数据自动持久化，防止误操作造成数据丢失。



### 5.4 易用性需求



1. Vue 前端界面布局简洁、导航清晰，操作流程符合用户使用习惯；
2. 功能按钮、表单输入有明确提示，降低使用学习成本。



### 5.5 可维护性需求



前后端均采用模块化开发，代码结构规范，基于 Trae CN 开发便于后期迭代，可快速扩展会员管理、线上支付、多房型价格策略等功能。



## 6 核心数据库设计（MySQL）



系统采用 MySQL 8.0 数据库，设计 5 张核心业务数据表，支撑所有功能开发：



1. **用户表（sys_user）**：存储住客 / 管理员账号、加密密码、姓名、手机号、角色类型、账号状态、注册时间；
2. **房型表（room_type）**：存储房型名称、床型、面积、基础价格、配套设施、排序序号、启用状态；
3. **房间表（room）**：存储房间号、楼层、关联房型 ID、房间当前状态、备注信息；
4. **订单表（booking_order）**：存储订单编号、用户 ID、房间 ID、入住离店日期、住宿天数、总金额、订单状态、入住人信息、下单时间；
5. **员工表（employee）**：存储前台 / 管理员账号、密码、姓名、联系方式、岗位权限、账号状态。



## 7 项目开发约束与假设



### 7.1 开发约束



1. **开发周期**：整体开发周期 8-10 周，包含需求分析、数据库设计、前后端编码、功能测试、文档整理；
2. **开发环境**：Windows/MacOS 系统，使用 Trae CN 作为唯一开发工具，前端 Vue3、后端 SpringBoot、MySQL 8.0；
3. **功能约束**：本版本为基础轻量化版本，暂不开发线上支付、会员充值、多分店管理、短信通知等高级功能。



### 7.2 开发假设



1. 所有用户具备基础浏览器上网操作能力，可独立完成注册、预订等操作；
2. 本地开发及运行环境网络稳定，可正常实现前后端接口联调；
3. 住客预留的个人身份、联系方式信息真实有效；
4. 酒店工作人员严格按照系统流程办理入住、退房业务。

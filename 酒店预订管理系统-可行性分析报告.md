# 可行性分析报告

## 酒店预订管理系统

---

## 一、项目需求分析概况

### 1.1 核心需求

本项目旨在为中小型酒店提供一个轻量级的在线预订管理系统，覆盖管理员（房型管理、员工管理、统计分析）、前台（入住/退房、房态看板）和住客（在线预订、动态价格查看、订单查询）三大角色。

### 1.2 需求规模估算

| 需求类别 | 规模 | 优先级 |
|----------|------|--------|
| 用户认证与权限管理 | 3种角色、JWT认证 | 高 |
| 房型与房间管理 | CRUD + 图片上传 + 详情页 | 高 |
| 预订订单管理 | 在线预订、入住、退房全流程 | 高 |
| 动态定价 | 房价日历、周末溢价、促销折扣 | 中 |
| 统计报表与图表 | ECharts 3种图表 | 中 |
| 审计日志 | AOP切面自动记录 | 中 |
| 登录安全 | 限流防暴力破解 | 中 |

### 1.3 需求可行性初步判断

- **功能需求可行性**：全部需求均基于成熟技术栈实现，无技术障碍。
- **性能需求可行性**：中小酒店并发量低（每日数百订单），单机部署完全满足。
- **安全需求可行性**：Spring Security + JWT + BCrypt 三道防线，满足基本安全要求。

## 二、可行性要素分析

### 2.1 技术可行性

| 技术要素 | 选型 | 成熟度 | 风险 | 可行性 |
|----------|------|--------|------|--------|
| 后端框架 | Spring Boot 3.2 | 成熟 | 低 | ✅ |
| 前端框架 | Vue 3 + Element Plus | 成熟 | 低 | ✅ |
| ORM | MyBatis-Plus 3.5 | 成熟 | 低 | ✅ |
| 数据库 | MySQL 8.0 / H2 | 成熟 | 低 | ✅ |
| 认证 | JWT (jjwt 0.12) | 成熟 | 低 | ✅ |
| 图表 | ECharts 5 | 成熟 | 低 | ✅ |
| AOP | Spring AOP | 成熟 | 低 | ✅ |

**关键技术路径**：(1) 使用 Spring AOP + 自定义注解实现零侵入审计日志；(2) 基于 ConcurrentHashMap 实现内存级登录限流，5次失败锁定15分钟。

### 2.2 经济可行性

| 分析维度 | 结论 | 说明 |
|----------|------|------|
| 开发成本 | 中 | 单人约2周迭代开发完成。 |
| 基础设施成本 | 极低 | 单服务器部署，MySQL 免费，无第三方付费API。 |
| API或服务调用成本 | 零 | 全部自建，无外部依赖。 |
| 维护成本 | 低 | 前后端分离，架构清晰，易于扩展和维护。 |

### 2.3 操作可行性

| 评估维度 | 结论 | 说明 |
|----------|------|------|
| 用户学习曲线 | 低 | 界面直观，与主流酒店管理软件操作逻辑一致。 |
| 安装部署 | 简 | 一个命令行启动后端，一个命令行启动前端。 |
| 首次使用引导 | 有 | 预置种子数据（房型、房间、账号），开箱即用。 |
| 错误恢复 | 可靠 | 全局异常处理器捕获所有异常并返回友好提示。 |

### 2.4 法律可行性

系统不涉及第三方API调用、不收集敏感个人信息（仅入住人姓名和手机号用于预订登记），密码经BCrypt加密存储，符合基本数据安全要求。系统为学习/演示项目，不涉及商业许可问题。

## 三、项目的设计方案

### 3.1 系统架构

```
浏览器 (Vue 3 + Element Plus)
    │  HTTP REST API
    ▼
Spring Boot 3.2 (8080端口)
    ├── SecurityConfig (JWT 过滤器)
    ├── Controller层 (30+ 个端点)
    ├── Service层 (业务逻辑 + AOP)
    ├── Mapper层 (MyBatis-Plus)
    │
    ▼
MySQL 8.0 (hotel_booking 库, 7张表)
```

### 3.2 技术方案

| 层 | 技术栈 | 方案要点 |
|----|--------|---------|
| 前端 | Vue 3 + Vite + Element Plus + ECharts | 组件化开发，Axios请求，Hash路由。 |
| 后端 | Spring Boot 3.2 + MyBatis-Plus | REST API，JWT无状态认证，AOP审计。 |
| 存储 | MySQL 8.0 (H2开发) | MyBatis-Plus自动建表，种子数据初始化。 |
| 安全 | Spring Security + BCrypt + 限流 | 角色权限控制，密码加密，登录防暴力。 |

### 3.3 核心模块设计

| 模块 | 功能 | 关键设计 |
|------|------|---------|
| 用户认证 | 登录/注册/JWT鉴权 | 三表设计（sys_user + employee），角色区分。 |
| 房型管理 | 增删改查 + 图片上传 | 8字段详细信息展示，独立详情页。 |
| 预订管理 | 在线预订、入住、退房 | 订单状态机（PENDING→CHECKED_IN→COMPLETED）。 |
| 房价日历 | 动态定价规则 | 按日计算价格系数，支持WEEKEND/HOLIDAY/PROMOTION。 |
| 审计日志 | 操作自动记录 | @OperationLog 注解 + AOP Around切面。 |
| 登录限流 | 防暴力破解 | IP+用户名维度，5次失败锁定15分钟。 |

## 四、结论

**结论：完全可行。**

- 技术栈全部成熟，无技术风险。
- 经济成本极低（零外部费用，仅需一台服务器）。
- 操作简便，用户学习成本低。
- 代码已通过18项单元测试，100%通过率。
- 系统架构清晰可扩展，适合后续持续迭代。
